# The Weekly Threat Brief: Sparx Systems Under Fire and Active Exploitation Trends

## TL;DR
*   **Active Exploitation:** CISA has confirmed active exploitation of a long-standing Apache ActiveMQ vulnerability.
*   **Sparx Systems Crisis:** A critical cluster of vulnerabilities in Sparx Pro Cloud Server allows unauthenticated SQL injection, credential exposure, and sensitive data leakage.
*   **Infrastructure Risks:** New high-severity flaws in Dell PowerProtect Data Domain require immediate attention to prevent command injection and log-based information disclosure.
*   **Patching Priority:** Prioritize patching of public-facing infrastructure (ActiveMQ/Sparx) over minor CMS plugin vulnerabilities.

---

## Active Exploitation & Critical Infrastructure
### CISA flags Apache ActiveMQ flaw as actively exploited [BleepingComputer]
CISA has issued an alert regarding a high-severity vulnerability in Apache ActiveMQ that is currently being exploited in the wild. Despite being patched earlier this month, the vulnerability—which existed undetected for 13 years—remains a primary target for threat actors looking to gain initial access or move laterally.

**Phalanx Thoughts:** This is a classic case of "patch gap" exploitation. Attackers are weaponizing long-dormant code paths. If you haven't audited your ActiveMQ instances for the latest patches, assume they are already compromised or being actively probed.

---

## Vulnerability Spotlight: Sparx Pro Cloud Server
A flurry of critical and high-severity CVEs has hit the Sparx Pro Cloud Server ecosystem, effectively turning it into a high-value target for any attacker with network visibility.

*   **[CRITICAL] CVE-2025-15625:** Unauthenticated execution of arbitrary SQL queries.
*   **[CRITICAL] CVE-2025-15624:** Plaintext storage of passwords when using OpenID authentication.
*   **[CRITICAL] CVE-2025-15623:** Unauthenticated exposure of sensitive system and personal information.
*   **[MEDIUM] CVE-2025-15622:** Sparx Enterprise Architect client reveals plaintext OAuth2 client secrets.

**Phalanx Thoughts:** The combination of unauthenticated SQLi and plaintext credential storage is a catastrophic failure of secure design. This isn't just a bug; it’s a skeleton key for your engineering documentation and intellectual property. Isolate these servers immediately if they are exposed to the internet.

## Infrastructure & Enterprise Security
### Dell PowerProtect Data Domain Vulnerabilities [CVE]
Two high-severity flaws were disclosed for Dell PowerProtect Data Domain:
*   **CVE-2026-23778:** Command Injection vulnerability in DDOS (Feature Release 7.7.1.0 through 8.5).
*   **CVE-2026-23775:** Sensitive Information Log Injection vulnerability.

**Phalanx Thoughts:** Command injection on a backup appliance is a "game over" scenario. If an attacker gains execution here, they don't just steal data—they can wipe your recovery points, effectively neutralizing your ransomware contingency plan.

---

## Web & Application Security
### Log Injection & CMS Flaws [CVE]
*   **CVE-2026-6494 (High):** Log injection in Aap-mcp-server allowing social engineering via the `toolsetroute` parameter.
*   **CVE-2026-6439 (Medium):** Stored XSS in the VideoZen WordPress plugin.
*   **CVE-2026-6451 (Medium):** CSRF in the "CMS für Motorrad Werkstätten" WordPress plugin.

**Phalanx Thoughts:** While these are lower on the totem pole than the infrastructure flaws, log injection (CVE-2026-6494) is often overlooked. Attackers use this to poison logs, potentially misleading incident responders or triggering malicious alerts in SIEM systems.

---

## Defender Takeaways

1.  **Immediate Patching for ActiveMQ:** If you run Apache ActiveMQ, verify your version against the latest CISA guidance immediately. Do not wait for your next maintenance window.
2.  **Isolate Sparx Infrastructure:** Given the unauthenticated SQLi and credential exposure flaws, verify that your Sparx Pro Cloud Server instances are not reachable from the public internet. If they must be remote-accessible, enforce strict VPN or Zero Trust Network Access (ZTNA) controls.
3.  **Backup Appliance Hardening:** Prioritize the Dell PowerProtect updates. Ensure that logging for these devices is ingested into a secure, read-only SIEM to detect potential log injection or command execution attempts.
4.  **Credential Rotation:** Following the disclosure of CVE-2025-15624 (plaintext password storage), assume any credentials used within the Sparx environment are compromised. Initiate a mandatory password reset for all affected accounts once the patch is applied.
