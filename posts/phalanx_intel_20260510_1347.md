*   **Critical RCE:** A pre-auth RCE in `xrdp` (CVE-2025-68670) demands immediate patching across your Linux infrastructure.
*   **Supply Chain Attack:** OceanLotus is actively poisoning PyPI repositories with "ZiChatBot" malware.
*   **Massive Data Breach:** ShinyHunters has reportedly exfiltrated data from 275 million Canvas (Instructure) users.

## Vulnerabilities & Exploits: The Perimeter is Crumbling

**CVE-2025-68670: RCE in xrdp** [Kaspersky]
A pre-auth Remote Code Execution vulnerability in the `xrdp` server component has been identified and patched. 
> **Phalanx Thoughts:** This is a "drop everything" scenario. Pre-auth RCEs in remote desktop protocols are the holy grail for initial access brokers. If you are running `xrdp` on any internet-facing Linux host, assume it has been scanned; audit your logs for anomalous connection attempts immediately.*

**Actively Exploited cPanel/WHM Bug** [Malwarebytes]
A critical flaw in the cPanel/WHM admin interface is currently being exploited in the wild, granting attackers full access to websites without requiring valid credentials.

**PhantomRPC: The "Won't Fix" Privilege Escalation** [Malwarebytes, Kaspersky]
Microsoft has opted not to patch "PhantomRPC," a technique that allows attackers to create a fake RPC server to escalate privileges on Windows.
> **Phalanx Thoughts:** When a vendor labels a privilege escalation vector as a "feature," they are essentially handing a roadmap to threat actors. Defenders must shift focus to behavioral monitoring of RPC calls rather than waiting for a patch that isn't coming.*

**Q1 2026 Vulnerability Trends** [Kaspersky]
The latest Kaspersky report highlights a continued reliance on established C2 frameworks by APT groups, reinforcing that while vulnerabilities are the entry point, the post-exploitation phase remains highly standardized.

## Malware & Supply Chain: Poisoning the Well

**OceanLotus Targets PyPI with ZiChatBot** [Kaspersky]
The OceanLotus APT has been caught uploading malicious wheel packages to the Python Package Index (PyPI) to deliver a dropper called "ZiChatBot."
> **Phalanx Thoughts:** This is a classic supply chain play. Developers pulling dependencies blindly are the primary vector here. If your CI/CD pipeline doesn't include strict dependency pinning and hash verification, you are essentially running unvetted code from the internet.*

**Financial Threats & Industrial Espionage** [Kaspersky]
*   **JanelaRAT:** A persistent financial threat continues to plague Latin American users with updated infection chains.
*   **Industrial Threats:** Q4 2025 data shows a steady, if unglamorous, evolution in how attackers target industrial automation systems.

**Mobile & Consumer Threats** [Malwarebytes, Kaspersky]
*   **WhatsApp:** New flaws discovered that allow for the delivery of disguised malware via malicious files. Update immediately.
*   **FakeWallet:** Over twenty malicious apps masquerading as crypto wallets were purged from the Apple App Store.

## Phishing: The "Legitimate" Threat

**Weaponizing Cloud Infrastructure** [Malwarebytes, Kaspersky]
Attackers are increasingly moving away from disposable domains and toward the abuse of trusted infrastructure. 
*   **Google AppSheet:** Being used to send phishing emails that bypass traditional security checks because they originate from Google’s own infrastructure.
*   **Amazon SES:** Threat actors are leveraging the high reputation of Amazon’s Simple Email Service to deliver BEC (Business Email Compromise) and phishing campaigns that sail past reputation-based filters.

> **Phalanx Thoughts:** We are witnessing the "legitimization" of phishing. When the delivery mechanism is a trusted cloud service, traditional IP/domain reputation filtering fails. Security teams must pivot to content-aware analysis and internal email authentication (DMARC/SPF/DKIM) combined with strict user behavioral heuristics.*

## Defender Takeaways

1.  **Patch the Edge:** Prioritize the `xrdp` and cPanel patches. These are not "wait until next month" vulnerabilities; they are active entry points for ransomware and data theft.
2.  **Sanitize the Pipeline:** If your developers use PyPI, implement a private repository proxy (like Artifactory or Sonatype Nexus) that scans for malicious packages before they reach your build environment. Stop pulling directly from public registries.
3.  **Assume Cloud Abuse:** Update your email security posture to handle "trusted" senders. If you rely solely on reputation-based filtering, you are blind to attacks coming from Amazon SES, Google, or Microsoft 365 tenants. Implement robust DMARC enforcement and train users to scrutinize the *content* of the email, not just the sender's domain.
4.  **Monitor RPC:** Since Microsoft isn't patching PhantomRPC, use EDR/XDR tools to alert on suspicious RPC server creation or unusual cross-process communication originating from the RPC subsystem.
