## Critical Flaws and Active Exploits Dominate the Threat Landscape

This week's intelligence digest highlights a flurry of critical vulnerabilities, with several open-source platforms taking a significant hit. Defenders should prioritize patching and reassess their exposure to actively exploited threats, including a maximum-severity RCE in Flowise and state-sponsored DNS hijacking.

### TL;DR

*   A maximum-severity RCE in **Flowise (CVE-2025-59528)** is now actively exploited in attacks. Patch immediately.
*   **ChurchCRM** has been hit with a staggering **19 critical vulnerabilities**, including unauthenticated RCE, API authentication bypasses, and numerous SQL injection and XSS flaws.
*   **OrangeHRM** faces **five critical vulnerabilities**, ranging from arbitrary file read via path traversal to improper access control and weak encryption.
*   **APT28 (Forest Blizzard)** is actively exploiting insecure **MikroTik and TP-Link SOHO routers** in a global DNS hijacking campaign.
*   Over 1,000 exposed **ComfyUI instances** are being targeted in a cryptomining and proxy botnet campaign.

### Critical Vulnerabilities & Active Exploits

#### Flowise RCE Under Active Exploitation

**Max severity Flowise RCE vulnerability now exploited in attacks** [BleepingComputer]
A critical remote code execution (RCE) vulnerability, tracked as CVE-2025-59528, in the open-source platform Flowise is now being actively exploited in the wild. Flowise, used for building custom LLM apps and agentic systems, presents a tempting target for attackers seeking initial access or to compromise AI-driven infrastructure.

**Phalanx Thoughts:** The immediate exploitation of a maximum-severity RCE in a platform like Flowise underscores the speed at which threat actors weaponize new disclosures. Defenders must prioritize patching any internet-facing Flowise instances, as successful exploitation grants attackers arbitrary code execution, potentially leading to full system compromise and data exfiltration.

#### ChurchCRM: A Litany of Critical Flaws

The open-source church management system, ChurchCRM, has been deluged with a staggering **19 critical vulnerabilities** in versions prior to 7.1.0 (and 7.1.1 for one specific XSS). This extensive list includes multiple avenues for remote code execution, authentication bypass, SQL injection, and cross-site scripting, painting a grim picture for unpatched deployments.

*   **CVE-2026-39337 - Unauthenticated RCE in Install Wizard** [CVE]
    A critical pre-authentication remote code execution vulnerability exists in ChurchCRM's setup wizard, allowing unauthenticated attackers to inject arbitrary PHP code during installation.
    **Phalanx Thoughts:** An unauthenticated RCE in an install wizard is as bad as it gets. This flaw provides a direct, low-effort path for attackers to gain full control over the server hosting ChurchCRM, making it the most severe of the reported issues. Organizations running ChurchCRM must ensure their installations are fully updated and that the install wizard is not publicly exposed post-setup.

*   **CVE-2026-39339 - API Authentication Bypass** [CVE]
    A critical authentication bypass vulnerability in ChurchCRM's API middleware allows unauthenticated attackers to access API endpoints.
    **Phalanx Thoughts:** This API bypass effectively nullifies security controls for critical backend functions, enabling attackers to interact with the application's core logic without legitimate credentials. It could be chained with other vulnerabilities or used to manipulate sensitive data.

*   **CVE-2026-39331 - API Authorization Bypass Allows Authenticated User to Deactivate, Modify, and Spam Arbitrary Families** [CVE]
    An authenticated API user can modify any family record's state without proper authorization by simply changing the `{familyId}` parameter in requests.
    **Phalanx Thoughts:** While requiring authentication, this authorization bypass allows low-privilege users to perform actions far beyond their intended scope, impacting data integrity and potentially leading to privacy breaches or service disruption.

*   **CVE-2026-39343 - SQL Injection in Event Type Editor (Admin)** [CVE]
    A SQL injection vulnerability exists in `EditEventTypes.php`, accessible to administrators, where the `EN_tyid` POST parameter is not sanitized.

*   **CVE-2026-39342 - SQL injection searchwhat parameter via QueryView.php** [CVE]
    The `searchwhat` parameter via `QueryView.php` with `QueryID=15` is vulnerable to SQL injection for authenticated users with access to Data/Reports > Query Menu.

*   **CVE-2026-39341 - SQL injection in ChurchCRM.0** [CVE]
    The application is vulnerable to time-based SQL injection due to improper input validation in `Reports/ConfirmReportEmail.php?familyId=`.

*   **CVE-2026-39340 - SQL Injection in PropertyTypeEditor.php via Incorrect Sanitizer Substitution** [CVE]
    A SQL injection vulnerability exists in `PropertyTypeEditor.php`, part of the administration functionality for managing property type categories.

*   **CVE-2026-39334 - Blind SQL injection in SettingsIndividual.php** [CVE]
    An SQL injection vulnerability was found in `/SettingsIndividual.php`, allowing authenticated users without specific privileges to inject arbitrary SQL.

*   **CVE-2026-39330 - Blind SQL injection in PropertyAssign.php** [CVE]
    An SQL injection vulnerability was found in `/PropertyAssign.php` for authenticated users with "Manage Groups & Roles" and "Edit Records" permissions.

*   **CVE-2026-39329 - Blind SQL injection in EventNames.php** [CVE]
    An SQL injection vulnerability was identified in `/EventNames.php` for authenticated users with "AddEvent" privileges via the `newEvtTypeCntLst` parameter.

*   **CVE-2026-39327 - SQL injection in MemberRoleChange.php** [CVE]
    An SQL injection vulnerability was found in `/MemberRoleChange.php` for authenticated users with "Manage Groups & Roles" permissions.

*   **CVE-2026-39326 - Blind SQL injection in PropertyTypeEditor.php** [CVE]
    An SQL injection vulnerability was found in `/PropertyTypeEditor.php` for authenticated users with `isMenuOptionsEnabled` role.

*   **CVE-2026-39325 - Blind SQL injection in SettingsUser.php** [CVE]
    An SQL injection vulnerability was found in `/SettingsUser.php`, allowing authenticated administrative users to inject arbitrary SQL statements.

*   **CVE-2026-39344 - Reflected XSS the login page through the 'username' parameter** [CVE]
    A Reflected Cross-Site Scripting (XSS) vulnerability exists on the login page due to lack of sanitization or encoding of the `username` parameter.

*   **CVE-2026-39338 - Blind XSS via Global Search – Administrative Cookie Session Exfiltration** [CVE]
    A Blind Reflected Cross-Site Scripting vulnerability exists in the search parameter accepted by the ChurchCRM dashboard, leading to administrative cookie session exfiltration.

*   **CVE-2026-39336 - Stored XSS from unescaped config values in HTML attributes** [CVE]
    A stored cross-site scripting issue affects Directory Reports form fields, Person editor defaults, and external self-registration.

*   **CVE-2026-39335 - Stored XSS via Unescaped data-* Attributes in Group/Family Controls** [CVE]
    Stored XSS exists in group remove control and family editor state/country, primarily an admin-to-admin path when writable entity fields are abused.

*   **CVE-2026-39333 - Reflected XSS in DateStart/DateEnd parameters in FindFundRaiser.php** [CVE]
    The `FindFundRaiser.php` endpoint reflects user-supplied input (`DateStart` and `DateEnd`) into HTML input field attributes without proper encoding.

*   **CVE-2026-39332 - Reflected Cross-Site Scripting (XSS) in GeoPage.php** [CVE]
    A reflected Cross-Site Scripting (XSS) vulnerability in `GeoPage.php` allows any authenticated user to inject arbitrary JavaScript.

*   **CVE-2026-39328 - Stored XSS in Social Profile Fields** [CVE]
    A stored cross-site scripting vulnerability exists in ChurchCRM's person profile editing functionality, allowing non-administrative users with "EditSelf" permission to inject malicious scripts.

**Phalanx Thoughts on ChurchCRM:** The sheer volume and severity of these vulnerabilities indicate deep-seated security issues within ChurchCRM prior to version 7.1.0. The combination of unauthenticated RCE, API bypasses, and numerous SQLi/XSS flaws means that any unpatched instance is at extreme risk of full compromise, data theft, or defacement. Organizations using ChurchCRM must prioritize upgrading to the latest secure version immediately.

#### OrangeHRM: Critical Data and Access Control Issues

OrangeHRM, a popular human resource management system, is affected by five critical vulnerabilities across versions 5.0 to 5.8, impacting data confidentiality, integrity, and access control.

*   **CVE-2026-39349 - OrangeHRM Uses AES-ECB for Sensitive Data Encryption Enables Pattern Disclosure** [CVE]
    OrangeHRM Open Source encrypts certain sensitive fields with AES in ECB mode, which preserves block-aligned plaintext patterns in ciphertext, enabling pattern disclosure.
    
**Phalanx Thoughts:** The use of AES-ECB for sensitive data is a critical cryptographic misstep. While not directly leading to RCE, it can allow attackers to infer patterns in encrypted data, potentially revealing sensitive information like salary ranges, employee IDs, or other structured data, undermining data confidentiality.

*   **CVE-2026-39348 - OrangeHRM is Missing Authorization Checks in AbstractFileController Subclasses Expose Job Specification and Vacancy Attachments** [CVE]
    OrangeHRM Open Source omits authorization on job specification and vacancy attachment download handlers, allowing authenticated low-privilege users to download sensitive attachments.
    **Phalanx Thoughts:** This authorization bypass allows unauthorized access to potentially sensitive HR documents, such as job specifications or candidate resumes. This could lead to data leakage and compliance violations.

*   **CVE-2026-39347 - OrangeHRM's Self‑Appraisal Submission of Admin Users Can Be Modified After Completion** [CVE]
    OrangeHRM Open Source accepts changes to self-appraisal submissions for administrator users after those submissions have been marked completed, breaking audit trails and data integrity.

*   **CVE-2026-39346 - OrangeHRM has Improper Access Control Allowing Access to Disabled Modules via URL Encoding** [CVE]
    OrangeHRM Open Source allowed authenticated users to bypass disabled-module access controls via URL-encoded request paths and access functionality of modules that should be inaccessible.

*   **CVE-2026-39345 - OrangeHRM Affected by Arbitrary File Read via Path Traversal in Email Template Loader** [CVE]
    OrangeHRM Open Source fails to restrict email template file resolution, allowing an authenticated actor who can influence email template paths to perform arbitrary file reads.
    **Phalanx Thoughts:** An arbitrary file read via path traversal can expose configuration files, source code, or other sensitive system files, providing attackers with valuable information for further exploitation or direct data exfiltration. Combined with other flaws, this could significantly escalate an attack.

**Phalanx Thoughts on OrangeHRM:** These vulnerabilities highlight a range of issues from weak cryptographic practices to fundamental access control failures. Organizations relying on OrangeHRM should prioritize patching to prevent sensitive data exposure, unauthorized access to modules, and potential information gathering by malicious insiders or compromised accounts.

### High-Impact Exploits

#### APT28 Exploits SOHO Routers in Global DNS Hijacking Campaign

**Russian State-Linked APT28 Exploits SOHO Routers in Global DNS Hijacking Campaign** [TheHackerNews]
The Russia-linked threat actor APT28 (aka Forest Blizzard) has been linked to a new campaign compromising insecure MikroTik and TP-Link routers since at least May 2025. The campaign, codenamed "Pawn Storm," modifies router settings to turn them into malicious infrastructure for DNS hijacking as part of a cyber espionage effort.

**Phalanx Thoughts:** APT28's targeting of SOHO routers for DNS hijacking is a classic but effective tactic for espionage. By controlling DNS resolution, the group can redirect victims to malicious sites, facilitate credential harvesting, or deliver malware. Organizations and individuals using these routers, especially those with remote workers, must ensure they are patched, have strong, unique passwords, and are configured securely.

#### ComfyUI Instances Targeted in Cryptomining Botnet Campaign

**Over 1,000 Exposed ComfyUI Instances Targeted in Cryptomining Botnet Campaign** [TheHackerNews]
An active campaign is targeting internet-exposed instances of ComfyUI, a popular stable diffusion platform, to enlist them into a cryptocurrency mining and proxy botnet. A purpose-built Python scanner continuously sweeps major cloud IP ranges for vulnerable targets, automatically installing malicious nodes via ComfyUI-Manager if no exploitable node is already present.

**Phalanx Thoughts:** The rapid weaponization of exposed AI/ML platforms like ComfyUI for resource-intensive activities like cryptomining is a growing trend. Defenders must ensure that any publicly accessible instances of such platforms are adequately secured, ideally not directly exposed to the internet, and regularly monitored for unauthorized activity.

### Medium Severity Vulnerabilities

#### Docker Engine Authorization Bypass

**Docker CVE-2026-34040 Lets Attackers Bypass Authorization and Gain Host Access** [TheHackerNews]
A high-severity security vulnerability, CVE-2026-34040 (CVSS score: 8.8), has been disclosed in Docker Engine. This flaw could permit an attacker to bypass authorization plugins (AuthZ) under specific circumstances, stemming from an incomplete fix for CVE-2024-41110.

**Phalanx Thoughts:** While rated medium in this feed, a Docker AuthZ bypass is significant. It could allow unauthorized access to Docker daemon functionalities, potentially leading to container escape and host compromise. Organizations using Docker Engine with AuthZ plugins should review their configurations and apply the necessary patches.

### Defender Takeaways

1.  **Patch Immediately for Active Exploitation:** Prioritize patching any Flowise instances to mitigate CVE-2025-59528. The active exploitation means time is of the essence.
2.  **Comprehensive Review and Upgrade for ChurchCRM and OrangeHRM:** If your organization uses ChurchCRM (prior to 7.1.0/7.1.1) or OrangeHRM (versions 5.0-5.8), a full security audit and immediate upgrade to the latest secure versions are non-negotiable. The breadth and depth of vulnerabilities in these platforms pose an existential risk.
3.  **Secure SOHO Routers and Network Edge:** For MikroTik and TP-Link SOHO router users, especially in remote work scenarios, ensure firmware is up-to-date, default credentials are changed, and strong, unique passwords are enforced. Implement robust DNS security measures and monitor for suspicious DNS queries.
4.  **Harden Public-Facing AI/ML Platforms:** Review the exposure of platforms like ComfyUI. Implement strict network access controls, avoid direct internet exposure where possible, and monitor for unusual resource consumption (e.g., CPU spikes indicative of cryptomining).
5.  **Strengthen Authorization and Authentication:** The recurring themes of authentication and authorization bypasses across multiple applications highlight the need for rigorous security testing, especially for custom applications and APIs. Implement multi-factor authentication (MFA) and principle of least privilege everywhere possible.
