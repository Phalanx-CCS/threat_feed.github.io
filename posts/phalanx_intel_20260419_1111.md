# The Weekly Burn: RCEs in the Wild and the WordPress Plugin Plague

### TL;DR
*   **Critical RCEs:** Protobuf.js and iTerm2 have critical vulnerabilities allowing for arbitrary code execution.
*   **WordPress Under Fire:** A wave of authenticated RCE and Stored XSS vulnerabilities hit multiple plugins, requiring immediate patching.
*   **Botnet Activity:** The Nexcorium Mirai variant is actively weaponizing EoL routers and DVRs.
*   **SAIL Library:** Multiple heap buffer overflows found in the SAIL image processing library.

---

## Critical Vulnerabilities & Exploits

**Protobuf.js Arbitrary Code Execution (CVE-2026-41242)** [Source: CVE, BleepingComputer]
A critical flaw in `protobuf.js` allows attackers to inject arbitrary code into "type" fields, which executes during object compilation. With proof-of-concept code already circulating, this is a high-priority item for any Node.js environment.

> **Phalanx Thoughts:** This is a supply-chain nightmare. Because `protobuf.js` is a foundational library for many JS-based microservices, the blast radius is massive. If your CI/CD pipeline or backend services use this, audit your dependencies immediately and force an update to 8.0.1 or 7.5.5.

**iTerm2 SSH Conductor Protocol RCE (CVE-2026-41253)** [Source: CVE]
Displaying a specially crafted `.txt` file in iTerm2 (through 3.6.9) can trigger RCE via DCS 2000p and OSC 135 data sequences.

**SAIL Image Library Heap Overflows (CVE-2026-40494, CVE-2026-40493, CVE-2026-40492)** [Source: CVE]
The SAIL cross-platform image library is suffering from multiple heap buffer overflows across its TGA, PSD, and XWD decoders. These are classic memory corruption issues stemming from missing bounds checks and type confusion.

## WordPress Plugin Vulnerabilities
The ecosystem remains a primary attack vector. We are seeing a pattern of authenticated (Contributor+) Stored XSS and RCE.

*   **CMP – Coming Soon & Maintenance Plugin:** Arbitrary file upload leading to RCE (CVE-2026-6518). **[Critical]**
*   **EMC Scheduling Manager:** Stored XSS via shortcode (CVE-2026-0868).
*   **Contextual Related Posts:** Stored XSS via `other_attributes` (CVE-2026-2986).
*   **Categories Images:** Stored XSS via `z_taxonomy_image` (CVE-2026-2505).
*   **Content Blocks:** Stored XSS via `content_block` (CVE-2026-0894).
*   **Flipbox Addon for Elementor:** Stored XSS via custom attributes (CVE-2026-6048).
*   **Page Builder Gutenberg Blocks:** Stored XSS via iCal feed (CVE-2026-4801).

> **Phalanx Thoughts:** The prevalence of "Contributor+" vulnerabilities highlights the danger of allowing untrusted users access to your CMS dashboard. If you aren't strictly auditing user roles and keeping these plugins updated, you are effectively handing keys to your server to anyone with a contributor account.

## Infrastructure & IoT Threats

**Mirai Variant "Nexcorium" Weaponizing EoL Hardware** [Source: TheHackerNews]
Threat actors are actively leveraging CVE-2024-3721 to hijack TBK DVRs and legacy TP-Link routers. The goal is to fold these devices into a Mirai-based botnet for DDoS operations.

> **Phalanx Thoughts:** This is the "zombie" problem in full effect. These devices are rarely patched, and their administrative interfaces are often exposed to the public internet. If you have legacy IoT gear on your network, isolate it behind a strict firewall or decommission it—it’s only a matter of time before it’s part of someone else’s botnet.

## Defender Takeaways

1.  **Prioritize the Protobuf Patch:** If you are running Node.js services, identify if `protobuf.js` is in your dependency tree. Update to the patched versions immediately.
2.  **Audit WordPress Roles:** Review your WordPress user list. If you have "Contributor" or "Author" accounts that aren't strictly necessary, delete them. These accounts are the primary entry point for the current wave of Stored XSS and RCE exploits.
3.  **Isolate Legacy IoT:** If you must keep EoL DVRs or routers, ensure they are not reachable from the public internet. Use a VPN or a jump box if remote access is required.
4.  **Airflow Hygiene:** If you manage Apache Airflow, the recent batch of vulnerabilities is significant. Move to version 3.2.0 to mitigate the risk of secret exposure and unauthorized DAG execution.
