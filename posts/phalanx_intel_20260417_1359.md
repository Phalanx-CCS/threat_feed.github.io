# The Weekly Threat Brief: Zero-Days, Infrastructure Risks, and the Death of Trust

## TL;DR
* **Zero-Day Emergency:** Critical zero-days in Microsoft and Adobe Reader are under active exploitation; patch immediately.
* **Active Exploitation:** CISA has confirmed active exploitation of a long-standing Apache ActiveMQ vulnerability.
* **Sparx Systems Crisis:** A critical cluster of vulnerabilities in Sparx Pro Cloud Server allows unauthenticated SQL injection and credential exposure.
* **Trojanized AI:** Attackers are weaponizing fake AI sites (Claude) and support pages to drop persistent malware like PlugX.

---

## Active Exploitation & Critical Infrastructure
### CISA flags Apache ActiveMQ flaw as actively exploited [BleepingComputer]
CISA issued an alert regarding a high-severity vulnerability in Apache ActiveMQ that is currently being exploited in the wild. Despite being patched, this 13-year-old vulnerability remains a primary target for initial access.

**Phalanx Thoughts:** This is a classic case of "patch gap" exploitation. Attackers are weaponizing long-dormant code paths. If you haven't audited your ActiveMQ instances for the latest patches, assume they are already compromised or being actively probed.

### April Patch Tuesday & Adobe Zero-Days [Malwarebytes]
Microsoft’s latest updates address 167 vulnerabilities, including two zero-days. Simultaneously, Adobe has patched a zero-day in Reader that was actively exploited in the wild—triggerable simply by opening a PDF.

**Phalanx Thoughts:** When a zero-day allows for system compromise via a simple document open, the "patch-as-soon-as-possible" mantra becomes a survival requirement. Prioritize Adobe Reader and Windows kernel-level components immediately.

---

## Vulnerability Spotlight: Sparx Pro Cloud Server
A flurry of critical and high-severity CVEs has hit the Sparx ecosystem, turning it into a high-value target for any attacker with network visibility.

* **[CRITICAL] CVE-2025-15625:** Unauthenticated execution of arbitrary SQL queries.
* **[CRITICAL] CVE-2025-15624:** Plaintext storage of passwords when using OpenID authentication.
* **[CRITICAL] CVE-2025-15623:** Unauthenticated exposure of sensitive system and personal information.

**Phalanx Thoughts:** The combination of unauthenticated SQLi and plaintext credential storage is a catastrophic failure of secure design. This isn't just a bug; it’s a skeleton key for your engineering documentation and intellectual property. Isolate these servers immediately if they are internet-exposed.

## Malware & Infostealers
### The Weaponization of AI and Support Brands [Malwarebytes/Kaspersky]
The malware ecosystem is defined by the weaponization of brand impersonation. 
* **Fake Claude Sites:** Malicious sites posing as Claude AI install PlugX malware.
* **Omnistealer:** A new "everything-stealer" using the blockchain to target cloud storage and crypto wallets.
* **JanelaRAT:** A financial threat specifically targeting users in Latin America.

**Phalanx Thoughts:** The shift toward trojanizing legitimate-looking AI tools is a masterclass in social engineering. Defenders must move beyond signature-based detection and implement strict application control (AppLocker/WDAC) to prevent unauthorized binaries from executing.

---

## Phishing & Data Breaches
### Booking.com Data Exploitation [Malwarebytes]
A Booking.com breach is providing scammers with high-fidelity data to conduct targeted social engineering against travelers.

**Phalanx Thoughts:** This is the "high-fidelity phishing" era. When an attacker knows your check-in date and room number, traditional "look for typos" advice fails. Organizations must verify payment requests through out-of-band communication—if a hotel asks for money via a chat link, call the front desk directly.

---

## Defender Takeaways

1.  **Immediate Zero-Day Patching:** Prioritize the Adobe Reader and Microsoft updates from this week. These are the most likely vectors for immediate attack.
2.  **Infrastructure Hardening:** Verify Apache ActiveMQ versions and update Dell PowerProtect appliances (CVE-2026-23778) to prevent command injection.
3.  **Sparx Isolation & Resets:** Isolate Sparx Pro Cloud Server instances from the public internet. Because of CVE-2025-15624, initiate a mandatory password reset for all affected accounts once patched.
4.  **Application Control:** With the rise of fake AI apps, rely on Allow-listing rather than Block-listing. If an application isn't approved, it shouldn't run.
